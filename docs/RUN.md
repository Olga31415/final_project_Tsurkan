# Запуск проекта через Conda

Пошаговая инструкция от создания окружения до получения результатов.

---

## Шаг 1. Открыть терминал с Conda

Запустите **Anaconda Prompt** (или любой терминал, где работает команда `conda`).

> Проверка: `conda --version` должна вернуть версию без ошибок.

---

## Шаг 2. Создать изолированное окружение

```bash
conda create -n mlqm python=3.12 -y
```

Что происходит:
- `conda create -n mlqm` — создаёт новое окружение с именем `mlqm`
- `python=3.12` — устанавливает Python 3.12 (стабильная версия, рекомендуется для Windows)
- `-y` — автоматически отвечает «да» на все вопросы установщика

Создание занимает 1–2 минуты.

---

## Шаг 3. Активировать окружение

```bash
conda activate mlqm
```

В начале строки терминала появится префикс `(mlqm)` — это значит, что все команды `python` и `pip` теперь работают внутри изолированного окружения, не затрагивая остальную систему.

---

## Шаг 4. Перейти в папку проекта

```bash
cd "C:\Users\QML\Documents\ПО\финальный проект"
```

> Замените путь на свой, если папка находится в другом месте.

Проверка — команда `dir` (Windows) или `ls` (macOS/Linux) должна показать файлы проекта: `README.md`, `pyproject.toml`, `requirements.txt`, папки `src/`, `tests/`, `docs/` и т.д.

---

## Шаг 5. Установить зависимости

```bash
pip install -r requirements.txt
```

Что происходит: pip скачивает и устанавливает все библиотеки из списка — FastAPI, Uvicorn, Pydantic, httpx, pytest.

```bash
pip install -e .
```

Что происходит: устанавливает сам пакет `ml_quality_monitor` в режиме «editable» (редактируемый). Это позволяет запускать `python -m ml_quality_monitor` без дополнительных настроек путей — Python будет находить пакет автоматически.

> Точка `.` в конце означает «текущая папка» — pip читает `pyproject.toml` и знает, где искать исходный код (`src/`).

---

## Шаг 6. Проверить, что тесты проходят

```bash
pytest
```

Ожидаемый вывод — все тесты зелёные, что-то вроде:

```
.......... passed in X.XXs
```

Если всё прошло — код рабочий, можно запускать.

---

## Шаг 7. Запустить сервер

> ⚠️ Шаги 7 и 8 выполняются в **двух отдельных** окнах терминала. Оба должны быть в папке проекта с активированным окружением `(mlqm)`.

В **первом** терминале:

```bash
python -m ml_quality_monitor
```

Сервер запустился, если видите строки:

```
INFO:     Application startup complete.
INFO:     Uvicorn running on http://0.0.0.0:8000
```

Проверка в браузере: откройте `http://127.0.0.1:8000/health` — должен вернуться ответ:
```json
{"status": "ok"}
```

Интерактивная документация API (Swagger UI): `http://127.0.0.1:8000/docs`

> Сервер работает, пока открыт этот терминал. Не закрывайте его.

---

## Шаг 8. Запустить демо-скрипт

Откройте **второй** терминал, активируйте окружение и перейдите в папку:

```bash
conda activate mlqm
cd "C:\Users\QML\Documents\ПО\финальный проект"
```

Запустите скрипт:

```bash
python examples/send_demo_data.py
```

Что делает скрипт:
1. Отправляет 300 пар (предсказание + факт) на сервер
2. Первые 150 — «здоровые» (5% ошибок), следующие 150 — «деградация» (35% ошибок)
3. Принудительно вызывает пересчёт метрик (`POST /aggregate`)
4. Печатает результаты в терминал

---

## Шаг 9. Прочитать результаты

Скрипт выведет два блока:

### Метрики (`Metric points`)

```json
{
  "model_name": "fraud-detector-v1",
  "samples": 300,
  "accuracy": 0.807,
  "f1": 0.807,
  "latency_p50_ms": 81.3,
  "latency_p95_ms": 144.3,
  "latency_p99_ms": 149.3,
  "window_start": "...",
  "window_end": "..."
}
```

| Поле | Что означает |
|---|---|
| `samples` | Сколько пар (предсказание + факт) попало в окно |
| `accuracy` | Доля правильных предсказаний (0.807 = 80.7%) |
| `f1` | Гармоническое среднее precision и recall |
| `latency_p50_ms` | Медиана времени ответа модели |
| `latency_p95_ms` | 95% запросов были быстрее этого значения |
| `window_start/end` | Временное окно, за которое считались метрики |

### Тревоги (`Alerts`)

```json
[
  {"metric_name": "accuracy", "metric_value": 0.807, "threshold": 0.9, "status": "pending"},
  {"metric_name": "f1",       "metric_value": 0.807, "threshold": 0.85, "status": "pending"}
]
```

| Поле | Что означает |
|---|---|
| `metric_name` | Какая метрика упала ниже порога |
| `metric_value` | Фактическое значение (0.807) |
| `threshold` | Порог (0.9 для accuracy, 0.85 для f1) |
| `status: pending` | Тревога в очереди; не доставлена, т.к. webhook не настроен |

Обе тревоги сработали потому что 0.807 < 0.9 и 0.807 < 0.85 — именно так и задумано в демо.

В первом терминале нажмите:

## Быстрый справочник команд

```bash
# Один раз: создать окружение
conda create -n mlqm python=3.12 -y

# Каждый раз при работе с проектом
conda activate mlqm
cd "C:\Users\QML\Documents\ПО\финальный проект"
pip install -r requirements.txt   # только при первом запуске или обновлении зависимостей
pip install -e .                   # только при первом запуске

# Терминал 1 — сервер
python -m ml_quality_monitor

# Терминал 2 — демо
python examples/send_demo_data.py
```

## Часто встречаемые проблемы

| Проблема | Причина | Решение |
|---|---|---|
| `ModuleNotFoundError: ml_quality_monitor` | Пакет не установлен | Запустить `pip install -e .` |
| `address already in use` (порт 8000 занят) | Старый процесс сервера не закрыт | Закрыть предыдущий терминал с сервером |
| `Connection refused` в демо-скрипте | Сервер не запущен | Сначала запустить шаг 7 |
| `conda: command not found` | Conda не в PATH | Использовать Anaconda Prompt |
